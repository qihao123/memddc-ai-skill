# MemDDC-Pro

## 核心全局规则（严格强制执行，全程生效）

### 1. 初始化阶段
项目无文档无DDD时，自动扫描全代码逆向生成全套工程文档 + DDD领域模型 + 长记忆结构化快照并压缩保存，同时识别和提取项目现有代码风格规范。

### 2. 后续所有迭代永久规则
每次新增功能 / 修改原有代码 / 重构模块时：
- 必须主动调用当前MemDDC-Pro Skill
- 自动读取项目已有的：完整工程文档 + DDD模型 + 历史压缩记忆快照 + 代码风格规范
- 全部喂给大模型作为前置上下文约束，绝不遗忘历史架构决策和代码风格要求

### 3. 修改完成后强制闭环
自动同步更新所有关联文档、DDD模型内容；
执行代码风格检查和统一；
然后重新执行【文档压缩 + 会话记忆压缩】覆盖更新快照；
保持所有资料永远最新、永远轻量化适配上下文。

## 工作流程

### 阶段1：初始化流程（仅首次使用自动触发）

1. **扫描项目结构**
   - 扫描项目目录、源码结构
   - 识别实体、服务、数据库逻辑
   - 分析依赖关系和模块边界
   - 识别和提取项目现有代码风格规范

2. **生成工程文档**
   - 在 `docs/` 目录生成全套架构文档
   - 生成业务文档、ER图、流程图
   - 创建AI轻量化压缩文档版本
   - 生成代码风格指南文档

3. **构建DDD领域模型**
   - 生成 `ddd-model.md` 完整领域模型
   - 定义限界上下文、聚合、实体边界
   - 识别领域服务和领域事件

4. **创建记忆快照**
   - 生成 `mem-snapshot.json` 全局压缩长记忆快照
   - 执行双层压缩：文档压缩包 + 记忆快照存档
   - 保存历史架构决策、约束规则和代码风格规范

### 阶段2：迭代修改通用流程（每次改代码/加功能时执行）

1. **加载已有资料**
   - 自动加载 `docs/` 目录下所有文档
   - 读取 `ddd-model.md` DDD领域模型
   - 加载 `mem-snapshot.json` 最新压缩记忆快照
   - 加载代码风格规范和指南

2. **注入上下文约束**
   - 将全部内容作为上下文注入大模型
   - 严格遵循历史架构决策不跑偏
   - 确保代码修改符合领域模型定义
   - 严格遵守代码风格规范

3. **执行代码修改**
   - 按DDD边界与原有业务契约完成新增/修改/重构代码
   - 验证修改合法性，确保不破坏现有架构
   - 执行代码风格检查和统一
   - 记录本次修改决策和变更原因

### 阶段3：修改后同步压缩闭环（不可省略）

1. **比对代码变更**
   - 检测本次代码修改涉及的范围
   - 识别受影响的模块、服务和实体
   - 检查代码风格一致性

2. **同步更新文档**
   - 更新对应的 Markdown 文档
   - 同步更新 `ddd-model.md` DDD模型定义
   - 如有必要，更新代码风格指南
   - 确保文档与代码保持一致

3. **代码风格统一**
   - 执行全项目代码风格检查
   - 自动修复代码风格问题
   - 验证代码风格一致性

4. **重新压缩存档**
   - 对最新全套文档执行轻量化压缩更新
   - 合并本次修改决策，更新记忆快照
   - 保存最新的代码风格规范
   - 二次压缩归档，覆盖原有快照

5. **验证一致性**
   - 保证代码、文档、DDD、记忆压缩四者强一致
   - 验证代码风格统一
   - 验证所有资料永远最新、永远轻量化

```

```

用户：MemDDC-Pro 按DDC契约迭代更新

## 注意事项

1. **首次使用**：确保项目目录存在且包含源码文件
2. **权限要求**：需要对项目目录有读写权限
3. **性能考虑**：大型项目初始化可能需要较长时间
4. **版本控制**：建议将生成的文档和模型文件纳入版本控制
5. **备份建议**：在执行重大修改前，建议备份现有文档和模型文件

## 故障排除

### 常见问题

1. **初始化失败**
   - 检查项目目录权限
   - 确保项目中包含足够的源码文件
   - 查看日志文件了解具体错误

2. **记忆加载失败**
   - 检查记忆快照文件是否存在
   - 验证文件格式是否正确
   - 尝试重新初始化项目

3. **文档更新失败**
   - 检查文档目录权限
   - 确保磁盘空间充足
   - 查看日志文件了解具体错误

4. **模型一致性问题**
   - 手动检查DDD模型与代码的一致性
   - 尝试重新生成模型文件
   - 验证代码修改是否符合领域模型定义
