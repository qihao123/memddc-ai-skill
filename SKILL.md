# MemDDC

## 技能信息

- **技能名称**: MemDDC
- **版本**: 1.0.0
- **作者**: qihao123
- **描述**: 项目文档管理与代码迭代智能工具，提供自动文档生成、DDD模型管理、记忆压缩和代码风格统一功能
- **触发关键词**: MemDDC, 加载记忆约束修改, 按DDC契约迭代更新
- **适用场景**: 复杂架构迭代、遗留系统持续重构、长期多人维护工程AI约束治理

## 核心功能

### 1. 初始化流程（首次使用自动触发）

- **全量代码扫描**：深度扫描项目目录、源码结构、实体、服务、数据库逻辑、API接口、配置文件等
- **详细文档生成**：自动逆向生成 docs/ 全套详细软件工程文档，包括：
  - 架构文档：系统架构、模块划分、依赖关系、部署架构
  - 业务文档：业务流程、用例分析、需求规约、用户故事
  - 技术文档：API接口文档、数据库设计、算法说明、技术选型
  - 开发文档：编码规范、开发环境、测试策略、部署流程
  - 图表文档：ER图、流程图、时序图、架构图
- **详细DDD领域模型**：自动生成 ddd-model.md 完整领域模型，包括：
  - 限界上下文：边界定义、上下文映射、上下文关系
  - 聚合：聚合根、实体、值对象、领域事件
  - 领域服务：服务定义、职责边界、协作关系
  - 领域规则：业务规则、不变量、约束条件
  - 战术设计：实体设计、值对象设计、领域服务设计
- **智能记忆压缩**：生成原始记忆文件并做第一次双层压缩：
  - 文档压缩包：保留完整详细文档
  - mem-snapshot.json 记忆快照：智能压缩核心信息，确保AI能快速理解

### 2. 迭代修改通用流程（每次改代码/加功能时执行）

- **快速记忆加载**：触发Skill后自动加载最新压缩记忆快照，无需读取原始代码和完整文档
- **智能上下文注入**：将压缩后的核心信息作为上下文注入大模型，确保严格遵循历史架构约束
- **精准代码修改**：按DDD边界与原有业务契约完成新增/修改/重构代码，减少上下文检索时间
- **实时验证**：在修改过程中实时验证代码是否符合领域模型和架构约束

### 3. 修改后同步压缩闭环（修改结束强制执行）

- **智能变更检测**：比对代码变更，识别影响范围和变更类型
- **文档同步更新**：同步更新对应详细 Markdown 文档与 DDD模型定义，确保文档与代码保持一致
- **智能压缩优化**：重新对最新全套文档执行轻量化压缩更新，保留核心信息，减少冗余内容
- **记忆快照更新**：合并本次修改决策，更新记忆快照并二次压缩归档，确保记忆快照始终反映最新状态
- **一致性验证**：保证代码、文档、DDD、记忆压缩四者强一致，避免信息偏差

## 文件目录规范

- `/docs` - 全量工程原始文档 & AI轻量化压缩文档
- `/ddd-model.md` - 项目完整DDD领域模型定义
- `/mem-snapshot.json` - 全局压缩长记忆快照文件

## 工作流程

### 初始化阶段

1. 扫描项目结构，分析源码文件
2. 生成完整工程文档（架构、业务、ER图、流程图）
3. 构建DDD领域模型，定义限界上下文、聚合和实体边界
4. 创建初始记忆快照并压缩存档

### 迭代修改阶段

1. 加载现有文档、DDD模型和记忆快照
2. 注入上下文约束，确保代码修改符合历史架构决策
3. 按照DDD边界和业务契约执行代码修改

### 闭环更新阶段

1. 比对代码变更，更新相关文档和DDD模型
2. 重新压缩文档和记忆快照
3. 确保所有资料保持最新且轻量化

## 配置选项

### 基本配置

- `projectRoot`: 项目根目录路径
- `docOutputPath`: 文档输出路径，默认为 `./docs`
- `dddModelPath`: DDD模型文件路径，默认为 `./ddd-model.md`
- `memSnapshotPath`: 记忆快照路径，默认为 `./mem-snapshot.json`

### 高级配置

- `scanDepth`: 目录扫描深度，默认为 5
- `compressionLevel`: 压缩级别，默认为 7（0-9）
- `contextInjectionThreshold`: 上下文注入阈值，默认为 80%
- `modelUpdateStrategy`: 模型更新策略，默认为 `incremental`

## 注意事项

1. **首次使用**：确保项目目录存在且包含源码文件
2. **权限要求**：需要对项目目录有读写权限
3. **性能考虑**：大型项目初始化可能需要较长时间
4. **版本控制**：建议将生成的文档和模型文件纳入版本控制
5. **备份建议**：在执行重大修改前，建议备份现有文档和模型文件

## 故障排除

### 常见问题

1. **初始化失败**
   - 检查项目目录权限
   - 确保项目中包含足够的源码文件
   - 查看日志文件了解具体错误
2. **记忆加载失败**
   - 检查记忆快照文件是否存在
   - 验证文件格式是否正确
   - 尝试重新初始化项目
3. **文档更新失败**
   - 检查文档目录权限
   - 确保磁盘空间充足
   - 查看日志文件了解具体错误
4. **模型一致性问题**
   - 手动检查DDD模型与代码的一致性
   - 尝试重新生成模型文件
   - 验证代码修改是否符合领域模型定义
