# MemDDC-Pro

## 技能信息

- **技能名称**: MemDDC-Pro
- **版本**: 1.0.0
- **作者**: H.Q
- **描述**: 项目文档管理与代码迭代智能工具，提供自动文档生成、DDD模型管理和记忆压缩功能
- **触发关键词**: MemDDC, MemDDC-Pro, 加载记忆约束修改, 按DDC契约迭代更新
- **适用场景**: 复杂架构迭代、遗留系统持续重构、长期多人维护工程AI约束治理

## 核心功能

### 1. 初始化流程（首次使用自动触发）

- 扫描项目目录、源码结构、实体、服务、数据库逻辑
- 自动逆向生成 docs/ 全套架构/业务/ER/流程图文档
- 自动生成 ddd-model.md 完整领域模型（限界上下文、聚合、实体边界）
- 生成原始记忆文件并做第一次双层压缩：文档压缩包 + mem-snapshot.json 记忆快照存档

### 2. 迭代修改通用流程（每次改代码/加功能时执行）

- 触发Skill后自动加载：已有所有文档 + DDD模型 + 最新压缩记忆快照
- 将全部内容作为上下文注入大模型，严格遵循历史架构约束不跑偏
- 按DDD边界与原有业务契约完成新增/修改/重构代码

### 3. 修改后同步压缩闭环（修改结束强制执行）

- 比对变更，同步更新对应 Markdown 文档与 DDD模型定义
- 重新对最新全套文档执行轻量化压缩更新
- 合并本次修改决策，更新记忆快照并二次压缩归档
- 保证代码、文档、DDD、记忆压缩四者强一致

## 文件目录规范

- `/docs` - 全量工程原始文档 & AI轻量化压缩文档
- `/ddd-model.md` - 项目完整DDD领域模型定义
- `/mem-snapshot.json` - 全局压缩长记忆快照文件

## 工作流程

### 初始化阶段

1. 扫描项目结构，分析源码文件
2. 生成完整工程文档（架构、业务、ER图、流程图）
3. 构建DDD领域模型，定义限界上下文、聚合和实体边界
4. 创建初始记忆快照并压缩存档

### 迭代修改阶段

1. 加载现有文档、DDD模型和记忆快照
2. 注入上下文约束，确保代码修改符合历史架构决策
3. 按照DDD边界和业务契约执行代码修改

### 闭环更新阶段

1. 比对代码变更，更新相关文档和DDD模型
2. 重新压缩文档和记忆快照
3. 确保所有资料保持最新且轻量化

### 核心模块

1. **项目扫描器**
   - 目录结构分析
   - 源码文件解析
   - 依赖关系识别

   - 领域服务提取
4. **记忆管理系统**
   - 文档压缩算法
   - 记忆快照生成
   - 历史决策记录
   - 上下文注入机制
5. **变更追踪器**
   - 代码变更检测
   - 文档同步更新
   - 模型一致性验证

### 执行逻辑

1. **初始化执行**
   - 检查项目结构

   - 创建记忆快照
2. **迭代执行**
   - 加载现有资料
   - 注入上下文约束
   - 执行代码修改

3. **闭环执行**
   - 检测代码变更
   - 更新相关文档
   - 同步DDD模型
   - 重新压缩存档

## 配置选项

### 基本配置

- `projectRoot`: 项目根目录路径
- `docOutputPath`: 文档输出路径，默认为 `./docs`
- `dddModelPath`: DDD模型文件路径，默认为 `./ddd-model.md`
- `memSnapshotPath`: 记忆快照路径，默认为 `./mem-snapshot.json`

### 高级配置

- `scanDepth`: 目录扫描深度，默认为 5
- `compressionLevel`: 压缩级别，默认为 7（0-9）
- `contextInjectionThreshold`: 上下文注入阈值，默认为 80%
- `modelUpdateStrategy`: 模型更新策略，默认为 `incremental`

## 注意事项

1. **首次使用**：确保项目目录存在且包含源码文件
2. **权限要求**：需要对项目目录有读写权限
3. **性能考虑**：大型项目初始化可能需要较长时间
4. **版本控制**：建议将生成的文档和模型文件纳入版本控制
5. **备份建议**：在执行重大修改前，建议备份现有文档和模型文件

## 故障排除

### 常见问题

1. **初始化失败**
   - 检查项目目录权限
   - 确保项目中包含足够的源码文件
   - 查看日志文件了解具体错误
2. **记忆加载失败**
   - 检查记忆快照文件是否存在
   - 验证文件格式是否正确
   - 尝试重新初始化项目
3. **文档更新失败**
   - 检查文档目录权限
   - 确保磁盘空间充足
   - 查看日志文件了解具体错误
4. **模型一致性问题**
   - 手动检查DDD模型与代码的一致性
   - 尝试重新生成模型文件
   - 验证代码修改是否符合领域模型定义
