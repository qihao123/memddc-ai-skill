## 项目核心思想

### 交流效率最大化

在软件开发过程中，人与大模型之间的交流存在显著的效率瓶颈：
- **传统打字交流**：速度约为 1-2 比特/秒，信息传递缓慢
- **文档驱动交流**：通过结构化文档传递信息，效率远高于直接对话

### 核心价值

MemDDC-Pro 的核心思想是**提高交流效率，减少大模型和人的低速沟通**。通过以下方式实现：

1. **自动文档生成**：将项目结构、业务逻辑、架构设计等信息自动转化为结构化文档
2. **DDD领域模型**：建立清晰的领域边界和业务契约，减少沟通歧义
3. **长记忆压缩**：将项目历史决策和约束规则压缩存储，确保大模型永远记得历史架构决策
4. **代码风格统一**：通过自动化工具确保代码风格一致性，减少因风格差异导致的沟通成本

### 工作原理

1. **初始化阶段**：扫描项目并生成完整文档、DDD模型和记忆快照
2. **迭代阶段**：自动加载现有资料作为上下文约束，确保代码修改符合历史架构决策
3. **闭环阶段**：自动同步更新文档和模型，保持所有资料最新且轻量化

## 如何使用

### 手动触发

通过以下关键词触发 MemDDC-Pro Skill：
- `MemDDC`
- `MemDDC-Pro`
- `加载记忆约束修改`
- `按DDD契约迭代更新`

### 自动调用设置

#### 编辑器自动调用配置

1. **VS Code 配置**：
   - 安装 TRAE 插件
   - 在 `settings.json` 中添加自动触发配置：

   ```json
   {
     "trae.autoInvokeSkills": [
       {
         "skillName": "MemDDC-Pro",
         "triggers": [
           "onFileSave",
           "onCodeEdit",
           "onProjectOpen"
         ],
         "conditions": {
           "fileExtensions": [".js", ".ts", ".jsx", ".tsx", ".java", ".py"],
           "projectTypes": ["node", "react", "angular", "vue", "java", "python"]
         }
       }
     ]
   }
   ```

2. **JetBrains 系列 IDE 配置**：
   - 安装 TRAE 插件
   - 在插件设置中启用自动触发：
     - 触发时机：文件保存、代码编辑、项目打开
     - 适用文件类型：所有源代码文件

3. **Sublime Text 配置**：
   - 安装 TRAE 插件
   - 在插件配置文件中添加：

   ```json
   {
     "auto_invoke": true,
     "skills": [
       {
         "name": "MemDDC-Pro",
         "triggers": ["on_save", "on_modify", "on_load"]
       }
     ]
   }
   ```

#### 项目级自动调用

在项目根目录创建 `.trae/config.json` 文件：

```json
{
  "autoInvoke": true,
  "skills": [
    {
      "name": "MemDDC-Pro",
      "triggers": [
        "onFileCreate",
        "onFileDelete",
        "onFileRename",
        "onBranchChange",
        "onMerge"
      ],
      "options": {
        "scanDepth": 5,
        "compressionLevel": 7,
        "autoUpdateDocs": true,
        "autoEnforceCodeStyle": true
      }
    }
  ]
}
```

## 技术特点

- **智能文档生成**：自动扫描项目并生成结构化文档
- **DDD领域建模**：建立清晰的领域边界和业务契约
- **长记忆管理**：压缩存储项目历史决策和约束规则
- **代码风格统一**：自动化代码风格检查和修复
- **上下文注入**：确保大模型严格遵循历史架构决策
- **闭环更新**：自动同步更新文档和模型，保持一致性

## 适用场景

- **复杂架构迭代**：确保架构决策的连续性和一致性
- **遗留系统重构**：快速理解现有系统并进行有序重构
- **长期多人维护**：保持代码风格和架构设计的一致性
- **跨团队协作**：通过结构化文档减少沟通成本

## 性能优化

- **增量扫描**：只处理变更的文件，减少扫描时间
- **并行处理**：同时处理多个文档生成任务
- **智能缓存**：缓存中间结果，提高重复操作效率
- **压缩算法**：使用高效压缩算法，减少存储和传输成本

## 故障排除

### 常见问题

1. **初始化失败**
   - 检查项目目录权限
   - 确保项目中包含足够的源码文件
   - 查看日志文件了解具体错误

2. **记忆加载失败**
   - 检查记忆快照文件是否存在
   - 验证文件格式是否正确
   - 尝试重新初始化项目

3. **文档更新失败**
   - 检查文档目录权限
   - 确保磁盘空间充足
   - 查看日志文件了解具体错误

4. **代码风格检查失败**
   - 检查代码风格配置文件是否正确
   - 确保代码风格检查器有执行权限
   - 查看日志文件了解具体错误
