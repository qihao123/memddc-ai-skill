## 模块划分

### 1. 游戏核心模块 (Core Module)

**职责**: 实现俄罗斯方块的核心游戏逻辑

**包含文件**:
- `tetromino.js` - 方块定义和变换
- `board.js` - 游戏板状态管理
- `collision.js` - 碰撞检测算法

**关键类**:
- `Tetromino` - 表示当前活动的方块
- `GameBoard` - 表示游戏板和固定方块
- `CollisionDetector` - 碰撞检测服务

### 2. 渲染模块 (Rendering Module)

**职责**: 负责游戏的视觉呈现

**包含文件**:
- `renderer.js` - Canvas 绘制逻辑
- `styles.css` - UI 样式

**关键功能**:
- 绘制游戏板网格
- 绘制当前方块
- 绘制已固定的方块
- 绘制分数和等级 UI

### 3. 输入模块 (Input Module)

**职责**: 处理用户键盘输入

**包含文件**:
- `input.js` - 键盘事件监听

**支持按键**:
- ← (左箭头) - 左移
- → (右箭头) - 右移
- ↓ (下箭头) - 加速下落
- ↑ (上箭头) - 旋转
- 空格 - 直接下落到底部

### 4. 游戏控制模块 (Game Control Module)

**职责**: 协调各模块，控制游戏流程

**包含文件**:
- `game.js` - 游戏主循环和状态管理

**关键功能**:
- 游戏主循环
- 方块生成
- 分数计算
- 游戏状态管理 (开始/暂停/结束)

## 依赖关系

```
game.js
├── tetromino.js
├── board.js
├── collision.js
├── renderer.js
└── input.js

renderer.js
└── board.js

input.js
└── game.js (回调)
```

## 数据流

```
1. 用户输入 → input.js
2. input.js → game.js (触发动作)
3. game.js → 更新 Tetromino/Board 状态
4. game.js → collision.js (验证移动)
5. game.js → renderer.js (重新渲染)
6. renderer.js → Canvas (显示更新)
```

## 技术选型

- **语言**: JavaScript (ES6+)
- **渲染**: HTML5 Canvas API
- **样式**: CSS3
- **无需外部依赖** - 纯原生实现

## 性能考虑

1. 使用 requestAnimationFrame 实现流畅动画
2. 游戏板使用二维数组存储，访问效率高
3. 碰撞检测使用提前返回优化
4. Canvas 绘制使用双缓冲避免闪烁
