# 俄罗斯方块网页游戏

一个基于 HTML5 Canvas 和原生 JavaScript 实现的俄罗斯方块游戏。

## 项目概述

本项目使用 memddc skill 的文档驱动开发流程进行构建，包含完整的 DDD 领域模型、架构文档和记忆快照。

## 生成过程

### 第一阶段：初始化文档结构 (memddc)

#### 1.1 创建 DDD 领域模型 (`ddd-model.md`)
定义了项目的领域驱动设计模型：

**限界上下文 (Bounded Contexts)**
- 游戏核心上下文 - 负责游戏逻辑
- 渲染上下文 - 负责视觉呈现
- 输入控制上下文 - 负责用户输入

**聚合 (Aggregates)**
- GameBoard 聚合 - 游戏板状态管理
- Tetromino 聚合 - 方块实体管理
- GameState 聚合 - 游戏状态管理

**领域实体**
- `Tetromino` - 俄罗斯方块实体
- `GameBoard` - 游戏板实体
- `Cell` - 格子实体

**领域服务**
- 碰撞检测服务
- 消行服务
- 分数计算服务

#### 1.2 创建架构文档 (`docs/architecture.md`)
设计了分层架构：

```
表现层 (Presentation)
├── index.html
└── styles.css

应用层 (Application)
└── game.js (游戏控制器)

领域层 (Domain)
├── tetromino.js (方块实体)
├── board.js (游戏板实体)
└── collision.js (碰撞检测)

基础设施层 (Infrastructure)
├── renderer.js (Canvas渲染)
└── input.js (输入处理)
```

#### 1.3 创建 API 文档 (`docs/api.md`)
定义了核心类的 API 接口：
- Tetromino 类 - 方块的移动、旋转、形状获取
- GameBoard 类 - 游戏板的验证、合并、消行
- Renderer 类 - 绘制游戏板、方块、UI
- InputHandler 类 - 键盘事件监听
- Game 类 - 游戏主控制器

#### 1.4 创建记忆快照 (`mem-snapshot.json`)
压缩存储项目核心信息：
- 项目元数据（名称、类型、语言）
- 文件结构
- 领域模型摘要
- 架构分层
- 游戏配置参数
- 方块类型和颜色定义
- 按键映射
- 代码风格规范

### 第二阶段：编写游戏代码

#### 2.1 方块实体 (`js/tetromino.js`)
实现了：
- 7种经典方块形状定义 (I, O, T, S, Z, J, L)
- 方块颜色配置
- Tetromino 类：
  - 位置管理 (x, y)
  - 旋转状态管理 (0-3)
  - 形状矩阵旋转算法
  - 移动方法 (左、右、下)
- 随机方块生成函数

#### 2.2 游戏板实体 (`js/board.js`)
实现了：
- Cell 类 - 格子状态（占用/颜色）
- GameBoard 类：
  - 二维网格初始化
  - 位置有效性验证（边界检测、碰撞检测）
  - 方块合并到游戏板
  - 满行检测与消除
  - 行下移算法
  - 游戏结束检测

#### 2.3 渲染器 (`js/renderer.js`)
实现了：
- Canvas 画布管理
- 游戏板绘制（已固定方块）
- 当前方块绘制
- 3D 效果绘制（高光、阴影）
- 网格线绘制
- 游戏信息绘制（分数、等级、行数）
- 下一个方块预览
- 暂停/游戏结束画面
- 颜色处理工具（变亮、变暗）

#### 2.4 输入处理器 (`js/input.js`)
实现了：
- 键盘事件监听
- 按键映射（方向键、空格、P、R）
- 按键重复控制（长按连续移动）
- 防止默认行为（页面滚动）

#### 2.5 游戏控制器 (`js/game.js`)
实现了：
- 游戏配置常量
- 分数计算表
- Game 类：
  - 游戏状态管理（ready/playing/paused/gameover）
  - 游戏主循环 (requestAnimationFrame)
  - 自动下落逻辑
  - 方块生成与管理
  - 分数、等级、行数统计
  - 速度递增算法
  - 各种操作处理（移动、旋转、硬降）
  - 墙踢 (Wall Kick) 实现
  - 游戏结束判定
  - 渲染协调

#### 2.6 样式文件 (`css/styles.css`)
设计了：
- 渐变背景
- 游戏容器布局
- 侧边栏信息面板
- 分数/等级/行数显示
- 下一个方块预览区
- 操作说明面板
- 开始界面样式
- 响应式设计（移动端适配）

#### 2.7 主页面 (`index.html`)
构建了：
- 游戏标题
- Canvas 游戏区域
- 开始界面遮罩
- 侧边信息面板
- 操作说明列表
- ES6 模块导入
- UI 更新逻辑

### 第三阶段：运行验证

1. 检查项目文件结构完整性
2. 启动 Python HTTP 服务器 (`python3 -m http.server 8080`)
3. 在浏览器中访问验证
4. 测试游戏功能：
   - 方块生成与移动
   - 旋转与墙踢
   - 消行与计分
   - 等级提升
   - 暂停/继续
   - 游戏结束与重开

## 游戏特性

### 核心玩法
- ✅ 7种经典俄罗斯方块
- ✅ 流畅的旋转与移动
- ✅ 智能墙踢系统
- ✅ 软降与硬降
- ✅ 自动消行

### 计分系统
| 消除行数 | 基础分数 |
|---------|---------|
| 1 行 | 100 × 等级 |
| 2 行 | 300 × 等级 |
| 3 行 | 500 × 等级 |
| 4 行 | 800 × 等级 |

### 等级系统
- 每消除 10 行升一级
- 下落速度随等级增加
- 最高速度限制

### 操作方式
| 按键 | 功能 |
|------|------|
| ← | 左移 |
| → | 右移 |
| ↓ | 加速下落（软降）|
| ↑ | 旋转 |
| 空格 | 直接落到底部（硬降）|
| P | 暂停/继续 |
| R | 重新开始 |

## 技术栈

- **渲染**: HTML5 Canvas API

## 开发方法

本项目采用 **文档驱动开发 (Document-Driven Development)** 方法：

1. **先设计文档** - 使用 memddc skill 生成完整的架构和领域模型
2. **后编写代码** - 严格按照文档定义实现功能
3. **保持一致性** - 代码与文档始终保持同步

这种方法的优势：
- 清晰的架构设计
- 明确的职责边界
- 易于维护和扩展
- 便于团队协作
