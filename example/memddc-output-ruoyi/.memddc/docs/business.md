# RuoYi-Vue 业务文档

## 系统定位

若依（RuoYi）是一款 **快速开发平台/后台管理系统脚手架**，目标是为中后台管理类应用提供完整的基础能力，让开发者专注于业务功能开发。

## 核心业务能力

### 1. 系统管理（Sys）

| 功能 | 业务说明 |
|------|---------|
| 用户管理 | 系统用户CRUD、状态控制、密码重置、分配角色岗位 |
| 角色管理 | 角色定义、权限分配（菜单权限+数据权限） |
| 菜单管理 | 菜单/按钮权限树、路由配置、图标、排序、可见性控制 |
| 部门管理 | 组织机构树、数据权限范围绑定 |
| 岗位管理 | 岗位字典、用户关联 |
| 字典管理 | 系统级字典（sys_dict_type/sys_dict_data），支持动态缓存 |
| 参数管理 | 运行时配置参数 |
| 通知公告 | 系统公告发布与阅读状态 |

### 2. 系统监控（Monitor）

| 功能 | 业务说明 |
|------|---------|
| 在线用户 | 查看当前Redis会话中的活跃用户，支持强制下线 |
| 定时任务 | 基于Quartz的Cron任务管理（增删改查、暂停/恢复、执行日志） |
| 数据监控 | Druid监控页面嵌入 |
| 服务监控 | 服务器CPU/内存/磁盘/JVM等运行状态 |
| 操作日志 | 用户操作行为审计（AOP自动记录） |
| 登录日志 | 登录/登出行为记录、异常登录分析 |

### 3. 系统工具（Tool）

| 功能 | 业务说明 |
|------|---------|
| 代码生成 | 根据数据库表结构自动生成前后端代码（Controller/Service/Mapper/前端页面） |
| 系统接口 | Swagger API文档浏览 |
| 表单构建 | 可视化表单设计器 |

## 业务实体关系

```
SysUser (用户)
 ├── N SysRole (角色)  via sys_user_role
 ├── N SysPost (岗位)  via sys_user_post
 └── 1 SysDept (部门)

SysRole (角色)
 ├── N SysMenu (菜单权限) via sys_role_menu
 └── N SysDept (数据权限范围) via sys_role_dept

SysDept (部门) ← 自关联树结构 (parent_id)
SysMenu (菜单) ← 自关联树结构 (parent_id)
SysDictType → N SysDictData
SysNotice → N SysNoticeRead
SysJob → N SysJobLog
```

## 关键业务规则

1. **数据权限规则**（DataScope）:
   - `1`: 全部数据
   - `2`: 本部门数据
   - `3`: 本部门及以下数据
   - `4`: 仅本人数据
   - `5`: 自定义部门数据

2. **用户状态**: `0=正常`, `1=停用`
3. **菜单类型**: `M=目录`, `C=菜单`, `F=按钮`
4. **操作状态**: `0=正常`, `1=异常`（记录操作日志）
5. **任务状态**: `0=正常`, `1=暂停`
6. **通知类型**: `1=通知`, `2=公告`
