# DDD 领域模型

## 限界上下文 (Bounded Contexts)

### 1. 系统管理上下文 (System Management Context)
**核心领域**: 用户、角色、权限、菜单、部门

#### 聚合根
- **SysUser (用户聚合根)**
  - 实体: SysUser
  - 值对象: LoginUser
  - 关联: SysDept, SysRole

- **SysRole (角色聚合根)**
  - 实体: SysRole
  - 关联: SysMenu, SysUser

- **SysDept (部门聚合根)**
  - 实体: SysDept
  - 特点: 树形结构，支持递归

- **SysMenu (菜单聚合根)**
  - 实体: SysMenu
  - 特点: 树形结构，与权限关联

#### 领域服务
- `ISysUserService`: 用户管理服务
- `ISysRoleService`: 角色管理服务
- `ISysDeptService`: 部门管理服务
- `ISysMenuService`: 菜单管理服务

#### 领域事件
- 用户创建/更新/删除
- 角色权限变更
- 部门结构调整

---

### 2. 字典管理上下文 (Dictionary Context)
**核心领域**: 数据字典、系统配置

#### 聚合根
- **SysDictType (字典类型)**
  - 实体: SysDictType
  - 值对象: SysDictData

- **SysConfig (系统配置)**
  - 实体: SysConfig

#### 领域服务
- `ISysDictTypeService`: 字典类型服务
- `ISysDictDataService`: 字典数据服务
- `ISysConfigService`: 配置服务

#### 缓存策略
- 字典数据缓存在 Redis
- 配置项缓存

---

### 3. 监控审计上下文 (Monitor Context)
**核心领域**: 操作日志、登录日志、在线用户、服务器监控

#### 实体
- **SysOperLog (操作日志)**
- **SysLogininfor (登录日志)**
- **SysUserOnline (在线用户)**

#### 领域服务
- `ISysOperLogService`: 操作日志服务
- `ISysLogininforService`: 登录信息服务

#### 特点
- 只写不读为主
- 大数据量考虑归档

---

### 4. 定时任务上下文 (Quartz Context)
**核心领域**: 定时任务调度

#### 聚合根
- **SysJob (定时任务)**
  - 实体: SysJob
  - 值对象: CronExpression

#### 领域服务
- `ISysJobService`: 任务管理服务

#### 技术实现
- 基于 Quartz 框架
- 支持并发/非并发执行

---

### 5. 代码生成上下文 (Generator Context)
**核心领域**: 代码自动生成

#### 聚合根
- **GenTable (生成表配置)**
  - 实体: GenTable
  - 值对象: GenTableColumn

#### 领域服务
- `IGenTableService`: 代码生成服务

---

## 领域规则与约束

### 用户管理
1. 用户账号唯一
2. 管理员账号不可删除
3. 用户与角色多对多
4. 用户与部门多对一

### 角色权限
1. 角色标识唯一
2. 数据权限范围：全部、自定义、本部门、本部门及以下、仅本人
3. 角色与菜单多对多

### 部门管理
1. 部门树不可循环引用
2. 删除部门需检查子部门
3. 祖级列表字段维护

### 菜单管理
1. 组件路径规范
2. 权限标识唯一
3. 菜单类型：目录、菜单、按钮

## 战术设计

### 实体设计
- 所有实体继承 `BaseEntity`
- 使用 JPA 注解进行验证
- 树形实体继承 `TreeEntity`

### 值对象
- `LoginUser`: 登录用户信息
- `MetaVo`: 路由元信息
- `RouterVo`: 路由信息

### 领域服务实现
- Service 接口定义契约
- ServiceImpl 实现业务逻辑
- 事务在 Service 层控制

### 仓储模式
- Mapper 作为仓储接口
- XML 映射 SQL
- 支持 MyBatis 动态 SQL
